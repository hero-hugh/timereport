- UI components live in `apps/web/src/components/ui/` and use `React.forwardRef` pattern with `displayName`
- Use `cn()` from `../../lib/utils` (or `../lib/utils` depending on depth) for Tailwind class merging

- Responsive layout pattern: use `md:hidden` / `hidden md:block` for mobile vs desktop views

- Lint check: `npx biome check <file>`

- Tests use `MemoryRouter` independently and are unaffected by the main router setup
- Run tests from repo root: `cd apps/web && npx vitest run` (working dir matters for git paths)
